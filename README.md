# DG-VDT

**Dynamic Graph-guided Vulnerability Detection and Traceability**

基于动态图引导的强化学习框架，用于以太坊智能合约的低延迟漏洞检测与攻击溯源。本文档梳理该方法的核心思路、表示设计与训练机制。

---

## 1. 问题与动机

在以太坊上同时做好两件事是困难的：**精确识别漏洞** 与 **可靠回溯攻击者**。现有两类主流路线各有结构性缺陷：

- **工具增强的大模型（tool-augmented LLM）**：通过监督微调（SFT）在大量检测轨迹上模仿学习。它在训练分布内表现尚可，但本质是表层统计模式的模仿，没有学到攻击的因果结构，因此一旦遇到结构上新颖的攻击（新型重入变体、跨链桥攻击等）就显著退化。
- **基于奖励信号的强化学习（RL）**：奖励多以"工具调用是否正确/输出格式是否合规"为准，缺乏对合约间交互拓扑的结构性理解。

一个典型的失败现象：SFT 模型把"环形资金流"这一表层拓扑当成恶意标志，于是在带环的重入样本上召回很高，但在**时间戳依赖**这类轨迹里没有环形价值转移结构的攻击上崩溃——判别性的信号（区块号/时间戳比较）其实就在轨迹里，模型却因为过度依赖表层拓扑而看不见。

**核心论点**：关系型图结构所编码的语义，远比"工具调用次数"或"输出格式"丰富；并且需要一条**渐进式课程**来调和开放域安全检测中"探索广度 vs. 匹配精度"的矛盾。

---

## 2. 总体思路

DG-VDT 把"漏洞检测 + 攻击者溯源"重新表述为**对执行轨迹的异构多图抽象之上的联合推理**，并用一套**图结构奖励**驱动纯强化学习（不做 SFT 预训练）。三个相互咬合的设计：

1. **EVM 插桩得到的多图表示**——把链上活动拆成三张语义互补的图；
2. **双阶段渐进式图引导奖励**——前期宽探索、后期严匹配，用 sigmoid 课程平滑过渡；
3. **GRPO 策略优化器**——在 LLM 骨干上用组相对策略优化训练。

针对三类结构上各不相同的漏洞：**重入（reentrancy）**、**短地址攻击（short-address）**、**时间戳依赖（timestamp-dependence）**。

---

## 3. 多图表示：从一条轨迹到三张图

每条 EVM 执行轨迹被分解为三张有向图，刻画区块链交互的不同维度：

| 图 | 记号 | 捕捉的语义 | 边的含义 |
|---|---|---|---|
| 资金流图 Fund Flow Graph | **FFG** $G_f$ | 交易的经济拓扑 | 地址间的 Ether 转移（金额、方向） |
| 合约创建图 Contract Creation Graph | **CCG** $G_c$ | 合约部署溯源 | 部署者 → 被部署合约 |
| 合约调用图 Contract Call Graph | **CaG** $G_\text{call}$ | 跨合约函数调用（opcode 粒度） | 调用顺序、调用类型 |

三张图合成一个统一快照 $G_\text{obs} = \{G_f, G_c, G_\text{call}\}$。

**节点类型**（地址分类学 + 辅助轨迹节点）：EOA（外部账户）、Contract（合约）、Miner（出块者），以及表示状态/操作码/分支/转账落点的辅助节点。
**边类型**（交互本体）：`ETH`（以太转移）、`CALL`（opcode 级调用）、`CREATE`（部署）、`SLOAD`/`SSTORE`（存储读写）。

这样的多图抽象，把"经济拓扑、部署来源、opcode 级调用语义"同时暴露出来——而逐合约（per-contract）方法或单图方法看不到这些。

---

## 4. 参考攻击签名 $G^*$

每一类漏洞都对应一个**规范参考图** $G^*$，作为该类攻击的最小判别结构。三类签名的结构规格：

- **重入 $G^*_\text{RE}$（7 节点 / 8 边）**：攻击者 EOA、受害合约、攻击者代理合约 + 余额账本状态节点。判别核心是 **FFG 中的资金环**（代理 → 受害 → 代理）叠加 **CaG 中对应的 CALL 重入边**。
- **短地址 $G^*_\text{SA}$（5 节点 / 5 边）**：攻击者 EOA、受害 ERC-20 合约、短地址收款方 + ABI 解码状态节点。判别核心是 **CaG 中 ABI 编码参数长度 < 32 字节**的 CALL 边——RGCN 编码器把这一截断参数长度映射成与正常 32 字节转账不同的判别性嵌入。
- **时间戳依赖 $G^*_\text{TD}$（6 节点 / 7 边）**：出块者账户（CCG 高出度枢纽）、彩票类合约、TIMESTAMP 操作码节点、条件分支节点、转账落点、余额状态节点。判别核心是 **CaG 中 `TIMESTAMP → TRANSFER` 的因果链**，并与 **CCG 中出块者账户的高出度枢纽**共现。

签名的选取采用**中位规模（median-size）**策略：在某类已确认为正例的图集合中，取节点数为中位数的连通图。相比取最小/最大/均值规模，中位数对图规模分布中的离群点更稳健——最小图会"欠约束"（判别力不足），最大图会"过约束"（罕见但合法的攻击变体匹配不上），中位数处于二者之间的最优点。

## 5. 图引导奖励机制

训练信号整体分解为：

$$R_\text{final} = R_\text{format} + R_\text{graph}$$

其中 $R_\text{format} \in \{0,1\}$ 是一个**前置门**：检查模型输出是否符合预定义语法
（`<vulnerability>TYPE</vulnerability>` 与 `<trace>ADDRESS</trace>`），用于稳定整个训练过程中的输出格式。信息量占主导的是 $R_\text{graph}$，通过下面的双阶段渐进机制工作。

### 5.1 Stage 1：宽图模式探索（软奖励）

早期训练由**嵌入相似度奖励**主导，目的是鼓励对图拓扑空间的广覆盖，而非过早特化。把观察快照 $G_\text{obs}$ 与参考图 $G^*$ 都用异构图编码器 $\Phi$ 投影到 $\mathbb{R}^{256}$，取余弦相似度：

$$r_\text{general} = -0.5 + \frac{\mathbf{z}_\text{obs} \cdot \mathbf{z}^*}{\lVert \mathbf{z}_\text{obs}\rVert\, \lVert \mathbf{z}^*\rVert}$$

对拓扑接近给予部分奖励，促使策略去探索结构变体（例如不完整的重入环），把精度的硬约束推迟到 Stage 2。

### 5.2 Stage 2：精确签名验证（硬奖励）

随着策略成熟，奖励切换到基于**子图同构**的严格二值判据。用 VF2 算法判断签名 $G^*$ 是否（在节点=地址类别、边=交互语义的标签约束下）出现在 $G_\text{obs}$ 内：

$$r_\text{strict} = \begin{cases} 1 & \text{若 } G^* \text{ 子图同构地嵌入 } G_\text{obs} \\ 0 & \text{否则} \end{cases}$$

二值的硬信号消除了部分给分的歧义，为"精确模式匹配"提供清晰的学习信号。

### 5.3 课程调度：sigmoid 平滑过渡

两个阶段通过连续的 sigmoid 计划混合：

$$R_\text{graph} = \sigma(t)\, r_\text{strict} + \bigl(1 - \sigma(t)\bigr)\, r_\text{general} + R_\text{collab} + R_\text{param}$$

$$\sigma(t) = \bigl(1 + e^{-k(t - m)}\bigr)^{-1}$$

$k$ 控制过渡陡度，$m$ 是中点步数。相比阶跃切换，sigmoid 斜坡避免了课程边界处的梯度突变。

### 5.4 两个辅助项

- **多图协同奖励** $R_\text{collab} = +0.3$：当多张图上**同时**检测到相互印证的异常时给予（例如 FFG 出现资金环、同时 CaG 出现对应的环形调用序列）。
- **参数误差惩罚** $R_\text{param} = -0.3$（每个错位的数值字段）：促使策略关注超出拓扑匹配之外的细粒度数值上下文。

---

## 6. 异构图编码器 $\Phi$

$\Phi$ 是关系图卷积网络（**RGCN**）的扩展，把图编码为 256 维向量；关系即边类型，节点特征为节点类型的 one-hot。

- **预训练**：在一份独立数据上用**图对比学习**目标训练。正样本对共享同一漏洞标签；难负样本取按余弦距离最相近的异类图（top-$k$）。
- **冻结**：预训练完成后，$\Phi$ 在整个 GRPO 过程中**保持冻结**。原因是：若 $\Phi$ 与策略 $\pi_\theta$ 联合更新，余弦相似度奖励会变成移动目标，放大梯度方差、引入奖励非平稳性。

---

## 7. 策略优化：GRPO

策略网络 $\pi_\theta$ 用 **Group Relative Policy Optimization (GRPO)** 优化。对每个查询 $q$，从参考策略采样一组 $N$ 个候选完成，目标为

$$\mathcal{J}_\text{GRPO}(\theta) = \mathbb{E}\left[\frac{1}{N}\sum_{i=1}^{N}\frac{1}{|o_i|}\sum_{t=1}^{|o_i|} \min\!\bigl(\rho_{i,t} A_i,\ \text{clip}(\rho_{i,t}, 1-\varepsilon, 1+\varepsilon) A_i\bigr)\right]$$

其中重要性比 $\rho_{i,t} = \pi_\theta(o_{i,t}\mid q, o_{i,<t}) / \pi_{\theta_\text{old}}(o_{i,t}\mid q, o_{i,<t})$，组内归一化优势 $A_i = (r_i - \bar{r}) / \text{std}(r)$。

关键设计：

- **不做 SFT 预训练**——纯 RL，沿用 R1-style 思路；
- **省略 KL 正则**——由 clip 边界隐式约束策略漂移；
- **退化批次防护**：二值的 $r_\text{strict}$ 在 GRPO 下可能出现"一组 $N$ 个样本奖励全相同"的退化批次，使组内归一化优势为零、梯度失效。此时保留来自 $R_\text{format}$、$R_\text{collab}$、$R_\text{param}$ 的梯度（它们在组内仍非常数），保证仍有非平凡的学习信号。

---

## 8. 输出格式与漏洞类别

策略需要输出符合如下 schema 的标记：

```
<vulnerability>TYPE</vulnerability>
<trace>ADDRESS</trace>
```

- `TYPE` ∈ { `reentrancy`, `short_address`, `timestamp` }
- `ADDRESS` 为 `0x` 开头的 40 位十六进制地址（攻击者溯源结果）

预测的漏洞类型决定了用哪一个参考签名 $G^*$ 去打分——这正是把"文本预测"锚定到"图结构"上的机制。

## 9. 记号速查

| 记号 | 含义 |
|---|---|
| $G_\text{obs}$ | 一个时间窗内的统一观察快照 $\{G_f, G_c, G_\text{call}\}$ |
| $G^*$ | 某类漏洞的规范参考攻击签名 |
| $\Phi$ | 异构图编码器（RGCN 扩展），图 → $\mathbb{R}^{256}$ |
| $\mathbf{z}_\text{obs}, \mathbf{z}^*$ | $G_\text{obs}$、$G^*$ 的图级嵌入 |
| $\sigma(t)$ | 课程 sigmoid，随训练步 $t$ 在两阶段间插值 |
| $\pi_\theta$ | 待优化的策略网络 |
| $N$ | GRPO 每个查询的候选完成数 |

---

## 方法要点一句话总结

> 把链上轨迹拆成资金流 / 创建 / 调用三张互补的图，用一个"先按图嵌入余弦相似度宽探索、再按 VF2 子图同构严匹配"的 sigmoid 课程奖励，去纯 RL（GRPO，无 SFT、无 KL）地训练一个 LLM，使其在结构异质的攻击模式上既能泛化、又能精确判别并回溯攻击者。
